[U-01] INSTALAR LIVE SERVER (para que los embeds de Twitch funcionen local)
       ► Los embeds de Twitch NO funcionan abriendo el HTML con doble clic
         (protocolo file://).  Necesitás un servidor local.
       ► Opción A (VS Code):
           1. Abrí VS Code → Extensions (Ctrl+Shift+X)
           2. Buscá "Live Server" de Ritwick Dey → Install
           3. Click derecho en index.html → "Open with Live Server"
       ► Opción B (Node.js global):
           npm install -g live-server
           En la carpeta del proyecto: live-server
       ► URL resultante: http://127.0.0.1:5500 — los embeds funcionarán ahí.

[U-02] CONSEGUIR LOS IDs REALES DE TUS VODS EN TWITCH
       ► Andá a twitch.tv/profe_pikashu/videos
       ► Abrí cada video que quieras mostrar
       ► En la URL vas a ver: twitch.tv/videos/XXXXXXXXXX  ← ese número es el ID
       ► Abrí js/video-config.js y reemplazá los IDs de ejemplo por los reales
       ► También podés agregar clips: clips.twitch.tv/NombreDelClip
         El "NombreDelClip" es el slug que ves en la URL del clip.

[U-03] INSTALAR NODE.JS (necesario para React, Vite y todo el stack moderno)
       ► Descargá desde: https://nodejs.org → versión LTS (la recomendada)
       ► Verificá la instalación:  node -v  y  npm -v  en la terminal
       ► Sin Node.js no se puede migrar a React ni usar Supabase, Clerk, Stripe.

[U-04] CREAR CUENTA EN SUPABASE (base de datos del proyecto)
       ► Registrate en: https://supabase.com  (gratis para proyectos pequeños)
       ► Creá un nuevo proyecto → guardá el Project URL y la anon key
       ► Esos datos van en un archivo .env que Claude te va a armar
       ► Sin Supabase: tienda sin backend, sin historial de compras, sin usuarios reales.

[U-05] CREAR CUENTA EN CLERK (autenticación)
       ► Registrate en: https://clerk.com  (gratis hasta 10.000 usuarios/mes)
       ► Creá una aplicación → habilitá login con Google y/o Twitch
       ► Guardá la Publishable Key → va en el .env
       ► Sin Clerk: el login/registro de la tienda es solo visual (sin funcionalidad real).

[U-06] CREAR CUENTA EN STRIPE (pagos de la tienda)
       ► Registrate en: https://stripe.com  (podés testear sin activar cuenta real)
       ► Modo test: usás tarjetas de prueba, no se cobra nada real
       ► Guardá la Publishable Key y la Secret Key → van en el .env
       ► Sin Stripe: el checkout es solo visual, no procesa pagos.

[U-07] REGISTRAR EL DOMINIO / CONFIGURAR DEPLOY
       ► Recomendado: comprar "andyazhtec.com" en NIC Argentina, Namecheap o similar
       ► Deploy gratis en: https://vercel.com  (conectás tu GitHub → deploy automático)
       ► Sin deploy: el sitio solo funciona en tu computadora.

► El embed de Twitch en la sección de videos usa window.location.hostname
  automáticamente para el parámetro "parent". No necesitás cambiarlo a mano
  cuando pases de localhost a producción.

► La tienda (store.html) comparte los mismos css/styles.css + css/store.css
  No hace falta duplicar los estilos base.

► El carrito de store.html guarda los datos en localStorage (clave: "atec_store_cart")
  Cuando Supabase esté conectado, esto se reemplazará por la DB.

► Los módulos de la página principal usan ES2022 sin transpilación.
  Si necesitás soporte para navegadores muy viejos (Edge <79, Safari <14),
  decíselo a Claude para agregar un polyfill o hacer la migración a Vite antes.
